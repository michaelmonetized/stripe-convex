## [bug] Pay component race condition
**Label:** bug

In Pay.tsx, clearCart() followed by addToCart() could cause race conditions. The clearCart() resets cart state, but if React batches these updates incorrectly, the addToCart could be lost.

Fix: Use a single setState call or pass items directly to checkout instead of mutating cart state.

## [architecture] Cart state not persisted
**Label:** architecture

Cart state is only stored in React state and resets on page refresh. For an e-commerce library, cart should persist via localStorage or URL params.

Consider:
1. localStorage persistence
2. URL state for shareable cart links
3. Optional server-side cart persistence via Convex
